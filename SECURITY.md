# 安全政策

Touchstone 是一个 AI PR 评审门禁系统——它读取代码、调用 LLM、在满足条件时参与合并决策。
因此它的安全性直接关系到接入它的每一个代码仓库。我们认真对待每一份安全报告。

## 支持的版本

| 版本 | 是否提供安全更新 |
| --- | --- |
| 1.0.x | ✅ |
| < 1.0 | ❌（预发布，请升级到 1.0.x） |

版本号以 `touchstone/__init__.py` 的 `__version__` 为准，可用 `python -m touchstone.run --version` 查看。

## 报告漏洞

**请勿通过公开 issue 报告安全漏洞。** 公开披露会在修复发布前把风险暴露给所有使用者。

请通过以下任一私密渠道报告：

- GitHub 私密漏洞报告：仓库 **Security → Report a vulnerability**（推荐）；
- 或邮件至维护团队（联系方式见仓库 README / 组织页面）。

报告请尽量包含：受影响的版本与组件、复现步骤或 PoC、你评估的影响面、以及（如有）建议的修复方向。

## 我们的响应承诺

| 阶段 | 目标时限 |
| --- | --- |
| 确认收到 | 3 个工作日内 |
| 初步影响评估与定级 | 7 个工作日内 |
| 修复或缓解方案 | 视严重程度，高危 30 天内、其余 90 天内 |
| 协同披露 | 修复发布后，与报告者协商时间公开致谢 |

我们遵循协同披露原则：在修复发布前不公开细节，发布时在 CHANGELOG 与安全公告中致谢报告者（除非报告者希望匿名）。

## 本系统特有的安全边界（供报告者参考）

Touchstone 的威胁模型有几处已知的关键边界，历史加固均围绕它们展开。理解这些有助于判断一个行为是否构成漏洞：

- **author 不可自证闭环**：PR 作者（含其 AI agent）不得在不修改代码的前提下，通过评论/申报让评审意见"销项"并触发自动合并。销项分 `VERIFIED`（机器复核，如签名复检不再命中）与 `CLAIMED`（author 自证 waived/split，需人核准）；后者绝不触发收敛或自动放行。**任何绕过此边界的路径都是高危漏洞。**
- **评审不可信不得伪装成通过**：当 LLM 评审失败（空响应、超上下文窗口被拒、引擎降级等）时，系统必须判定为"不可信"并阻止假收敛/自动放行，而非把"0 建议"呈现为"审过没问题"。**任何让静默故障通过为绿灯的路径都是高危漏洞。**
- **凭据隔离**：执行不可信 PR 代码的环境（verify_execute）必须零凭据——无 secret、`.git/config` 无 token、job 权限降至 `contents:read`。**任何让 PR 代码窃取到凭据或伪造门禁状态的路径都是高危漏洞。**
- **确定性核对不打折**：契约/安全/危险代码扫描（SEC-*/DANGER-* 等）跑全文 diff，不受显示截断影响。**任何让确定性检查被 diff 裁剪或输入构造绕过的路径都是漏洞。**
- **权威状态只信机器人自己**：收敛清单的权威状态从受信评论（机器人自己发的、经 `loop.trusted_bodies` 过滤）读取。**任何让 author 伪造权威 marker 影响判定的路径都是漏洞。**
- **verify 的"绿"是攻击者可影响信号，结构上不承重**：`verify-result.json` 由执行 PR 代码的零密 job 产出，恶意 PR 可伪造其内容（含 `passed=true`）。系统据此把它设计为非承重输入：总闸只"折入"（插件之一）、伪红只伤攻击者自己；自动放行需九闸 AND（含与 PR 内容无关的作者信任闸）、`author_proposed` 规格的绿不算通过、畸形值经 `_truthy` fail-closed。**任何让 verify 绿单独放行代码的配置或改动都是漏洞。**
- **自动放行有作者身份闸**：`autonomy.author_trusted` 按 GitHub `author_association`（默认 OWNER/MEMBER/COLLABORATOR，均需仓库侧授予）+ 显式 login 白名单判定；产物缺作者出处即不放行。**任何让不受信作者（fork 陌生人）绕过该闸进入自动合并的路径都是漏洞。**

## 不属于漏洞的范畴

- LLM 因合法原因返回空建议（小 PR 无问题）——系统会如实标注"0 发现"并溯源，这是预期行为；
- 顾问式评审的 AI 意见误报——评审是 advisory，不拦合入，人有最终决定权；误报请通过反馈改进，不按漏洞处理；
- 依赖库自身的漏洞——请报告给对应上游；若 Touchstone 的用法放大了其影响，则可报告。
