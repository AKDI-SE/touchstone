详细设计 · Phase 2 学习回路

Touchstone · 自进化评审学习回路

在 advisory-only（只给评审意见、合入按钮始终人点）定位下，复用开源 PR-Agent 当评审器，用人审裁决当奖励，TF-GRPO 把经验固化成注入提示词的 token prior，govern 退役有害经验——无训练、无权重、自托管。评审质量随真实使用持续改进，决策权始终在人。

定位 advisory-only · 人保留合入权 学习 训练-free（上下文空间，无权重） 奖励 人审裁决（采纳/改动/噪声） 部署 自托管 · 公网/自托管端点 · 冻结模型

01

目的

去掉自动合并后，唯一目标变成「高信噪比的评审意见」——而 AI 评审工具最常见的失败方式是规则越堆噪声越大、最终被一线开发弃用。本回路要的，是让评审质量能随真实使用自我改进，且不靠训练权重（数据稀缺、无训练基建、要可控可回退）。

做法：把「评审」与「学习」彻底解耦。评审走成熟开源（PR-Agent），学习是一条离线、周期性的回路，把「人最终采纳/忽略了什么」蒸馏成自然语言经验，回注评审器的提示词。这恰好把原设计里一直只是指针的「Phase 2 经验库/TF-GRPO」落到实处，且复用你已有的 calibrate 与 govern。

03

TF-GRPO 机制（Touchstone 自己的核心）

评审引擎复用了开源的 PR-Agent，所以 Touchstone 真正自己的、有差异化价值的部分，是让评审随真实使用自我改进这条回路——而 TF-GRPO 就是它的引擎。本节把这块机制写细。它直接取自 Training-Free Group Relative Policy Optimization（arXiv 2510.08191，2025-10），核心思想是：策略模型一概冻结、不更新任何权重，改为维护一份外部的"经验知识"当作注入提示词的 token prior；每一步对一个查询并行采样一组输出，用奖励打分，但不计算数值优势做梯度上升，而是让大模型内省这一组输出、提炼出"语义优势"（一段自然语言经验），累进地更新那份经验知识。

诚实交代（命名与范围）：本方法借鉴 GRPO 的 分组采样 + 冻结策略 + 经验作 token prior，但用「让大模型内省高分 vs 低分评审、提炼自然语言经验」替代了 GRPO 的数值组内相对优势——这是 Touchstone 的改写，并非完整复现 GRPO。代价：只取 top/bottom 少数样本做内省，方差比归一化优势更高。与 arXiv 2510.08191 的具体偏差，以原论文为准。

3.1 概念映射（从论文到 PR 评审）

论文概念在 Touchstone 里是什么
冻结策略 πPR-Agent（旗舰模型 + 提示词），参数始终不动
经验知识 E（token prior）经验库：一组自然语言经验，注入 PR-Agent 的 extra_instructions
查询 q一个历史 PR——已合入、已有人审裁决，用作"最小真值集"
一组 rollout {o₁…o_G}在当前经验库 E 下，对同一个 PR 生成的 G 份评审
奖励 r_i = R(q, o_i)第 i 份评审与"人实际采纳了什么"的吻合度：命中人关心的发现得分、报出人忽略的噪声扣分
数值优势 (r_i−mean)/std不用——本方法不做梯度更新
语义优势让旗舰模型内省整组评审：为什么高分的挑对了、低分的挑偏或挑漏，把差异提炼成按"仓·栈·发现类型"归类的经验
多轮(epoch)在最小真值集上离线多轮迭代，经验库逐轮累积、修正

3.2 离线过程（一轮 epoch 内，对每个历史 PR）

把上图写成步骤：对最小真值集里的每个历史 PR q——① 在当前经验库 E 下，让 PR-Agent 生成 G 份评审；② 用这个 PR 的人审真值（人最终改了/采纳了哪些点）给每份评审打分 r_i，命中人关心的加分、报出人忽略的扣分；③ 把整组带分数的评审交给旗舰模型内省，对比高分与低分，说清"在这个仓/栈/发现类型上什么该挑、什么别挑"，产出候选经验；④ 候选经验经去重与冲突消解后并入经验库，但先入候选池、不立即生效。多轮重复，经验库逐步收敛。

3.5 忠于论文、为 PR 评审做的改写（与诚实交代）

借自论文策略冻结、对查询分组采样一组输出、经验作 token prior 注入、多轮迭代。
Touchstone 的改写（非论文原貌）用「LLM 内省 high/low reward 的评审 → 自然语言经验」替代 GRPO 的数值组内相对优势——这是本系统的自创，不是论文机制。代价：仅取 top/bottom 少数样本内省，方差高于归一化优势。
关键改写奖励改为离线：PR 评审拿不到每次 rollout 的即时人反馈，所以不在线打分，而是用历史已合 PR 的人审真值离线评分（这正是 calibrate 已经在算的）。经验带 repo/stack 字段；store 每仓一份，去重键 (kind:finding_type) 仓内唯一（多仓共享同一 store 时才需把 repo/stack 纳入键）。且只对 PR-Agent 源的发现产经验；确定性 contract_check 永不进经验库、不受其影响，作为经验出问题时的固定基准。
实现与验证（诚实）本机制已实现，离线单元测试覆盖 plumbing——rollout_reviews / score_review / distill_semantic_advantage / _distill_via_llm，旗舰模型经注入的 llm，离线以假 llm 验步骤串联 / JSON 解析 / protected / locked。未实证的部分：蒸馏出的经验是否真能提升采纳率，靠 shadow A/B（graduate）事后验证，且需先有注入历史才生效——其端到端效果离线测不了（测不了 LLM 质量）。数据管线已接通：build_ground_truth 从 GitHub 人审裁决重建真值集；aggregate_ab 按 result marker 的 injected_types 自动切 A/B 两臂喂 graduate（无需手填 --ab-results）；多轮 epoch 用 active+candidate 工作库逐轮重条件（真迭代）。奖励信号：thread_findings 已过滤 wontfix（人不采纳的不当正例）。安全上坚持稳健上线：经验只调建议、不碰合入门禁，新经验先 shadow A/B 达标才启用（见下节）。唯一运行期依赖是一个参数固定的旗舰模型端点（①生成与③内省用它；端点可靠性见下节坑 1）。
已知上限① 发现类型天花板：human_adopted ⊆ touchstone 自己挑过的类型，故回路只能调既有类型的强调/压制，学不出"人关心但 touchstone 没挑过"的新类型；突破需把人写的评审评论也当 ground truth（未来扩展）。② 奖励粒度粗：finding_type 集合级匹配（如 PRA-SECURITY），hits/noise 都偏粗。③ bootstrap：candidate→active 需"注入臂"有数据，冷启动靠 seed 种子注入积累后才自动达标。

04

三个坑与应对

坑 1 · 自定义/自托管端点可靠性（前置风险） TF-GRPO 要跑大量 LLM 调用（分组 rollout × 多轮），PR-Agent 在自定义/自托管端点上有已知问题（配置自定义端点仍回退硬编码模型）。应对：把"PR-Agent 在你的自定义端点验证可正常运行"列为第一道验证关，不通则整条学习回路不上；且因评审/学习解耦，端点抖动时学习停摆、评审降级仍可跑（沿用上一版经验库）。
坑 2 · 奖励有偏（人审盲区） 奖励是"人采纳/合入"，噪声且有偏——TF-GRPO 会优化成"人爱看的"而非"对的"，可能放大人审盲区。应对：(a) advisory-only 下后果可控（不进合入门禁）；(b) 经验只影响 LLM 建议、不稀释确定性 contract_check——确定性检查不受经验影响，作为固定基准；(c) govern 退役高噪声/低采纳经验 + 人定期抽检经验库；(d) 新经验先入"建议池"、需达标才转 active。
坑 3 · 经验偏离的应对（shadow 先行） 经验由模型从人的采纳里蒸馏，难免有偏；为防一条坏经验拉低评审，新经验默认不直接注入，先离线对最近 PR 做 A/B（注入 vs 不注入）比采纳率，达标才转 active；小步、可回退；每条经验留来源 PR 与证据，可单条停用。

05

与现有件的映射 · 落地次序

映射

  • 直接复用：calibrate（奖励）、govern（退役/门控）——本就为标定与降噪而建，恰是 TF-GRPO 需要的奖励源与控制层。
  • 可替换/并存：你自研的委员会 orchestrator.py 可被 PR-Agent 取代，或两者并存（PR-Agent 出主评审、你的确定性检查与 repo-context 作补充）。
  • 保留作基准：contract_check 确定性检查不受经验影响，是经验出问题时的固定基准。

落地次序（每步可回退）

  1. 验证端点：PR-Agent 在你的自定义端点验证可正常运行（坑 1 的关）。
  2. 挂 advisory、积累奖励信号：PR-Agent 在活跃仓库真跑两周，calibrate 积累人审裁决信号。
  3. 离线进化 + shadow：周期跑 TF-GRPO 产候选经验，govern 门控，A/B 比采纳率（坑 2/3）。
  4. 达标启用注入：经验库转 active 注入 PR-Agent；持续由 govern 退役、人抽检。
实现状态（TF-GRPO + 可插拔蒸馏器）touchstone/learning_loop.py + 离线测试（共 228 全绿）。已实现：经验库 JSON 读写；训练-free 计数蒸馏 distill_candidates；TF-GRPO 全流程 rollout_reviews / score_review / distill_semantic_advantage / _distill_via_llm（旗舰模型经注入的 llm 调用，离线以假 llm 覆盖测试；确定性 contract 检查不进经验，坑 2b）；graduate（shadow A/B 达标 candidate→active）；retire（govern 式退役）；render_injection（仅 active → PR-Agent extra_instructions，明文标 advisory、不进门禁）；单条 disable 可回退；main 入口按是否提供真值集在 TF-GRPO 与计数式之间选择。生产前置：一个参数固定的旗舰模型端点（LLM_BASE_URL / LLM_API_KEY / TOUCHSTONE_FLAGSHIP_MODEL）+ 一份历史 PR 真值集（采什么数据、如何从 GitHub 重建见 data-collection-design.html）；端点可靠性见坑 1。蒸馏器可插拔：distill(ctx, name) 按名分发、register_distiller 注册自有实现（不必改本文件），_distill_via_llm 的 rollout/score/distill_advantage 三步可注入替换。
一句话一条无训练的自进化评审回路：复用开源评审器、用已有标定当奖励、用 TF-GRPO 把"这个仓该挑什么、别挑什么"固化成经验、用 govern 退役坏经验——而人始终持有合入决定权。

06

人能提供什么（人类输入入口）

training-free 的一个独特好处：经验就是一段人能直接读、直接写的自然语言，策略模型又冻结不动，所以人不只能提供数据让它学，还能直接读写它学到的东西——这是和权重式 RL 最不同的地方（权重式 RL 把数据变成模型权重，学到的东西人读不到、也改不了）。下面列出人能提供的输入及其落点。

人能提供是什么落点（已实现 / 设计）
手写经验种子资深工程师直接写规则（"本仓 @Transactional 标在 private 方法上必报"），入库当 active 或 candidate；解决冷启动seed_experience(source=human)（已实现）；也可写进 best_practices.md / standards.yaml
审校候选经验读学习回路产出的 candidate，改措辞、合并重复、纠正过度泛化、定作用域、批准或否决candidate→active 生命周期 + disable（已有）；审校界面（设计，建议走 GitHub 评论批准，延续不建库）
逐条显式反馈对每条意见给 👍/👎、标"误报/已知/漏报"、改风险等级——比自动推断更干净的奖励信号采纳证据最高可信度等级（见 data-collection-design）
奖励设计命中/噪声/漏报的权重、"算采纳"的判定、这个仓更看重精确率还是召回、标杆评审TOUCHSTONE_W_NOISE / W_MISS + score_review 权重参数（已实现）；标杆评审（设计）
